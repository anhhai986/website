リリースノート

2.4.7リリースノート

2017年11月12日

改善

  • sc-contentセクションを多く含むページでのパフォーマンス改善のために最適化が施されました

バグの修正

  • 編集モードでのページURLの編集可能URLへの変換
  • ファイルアップロードのURLを正しく設定
  • ヘッダーエレメント関連の問題対処のためにsc-contentクラス を設定
  • 公開後のメニュー更新問題への対処

2.4.6リリースノート

2017年10月30日

改善

  • バックグラウンドでのコンテンツ保存とページ公開の最適化
  • 公開時間の圧倒的短縮
  • 1回のページ読み込みでページ処理を実行（以前は全てのページはログインで利用可能となっていた）
  • コンテンツ処理の遅延原因となっていたディペンデンシライブラリの削除または書き換え
  • 内部PHPウェブサイト警告がログに記録されるものの、表示は解消

バグの修正

  • クロームブラウザの編集モードスクロールの問題を解消

2.4.5リリースノート

2017年6月15日

改善

  • クロアチア語(hr)への対応を開始
  • ページマネージャー：新規作成ページは公開後も選択状態に
  • ページマネージャー：ページの追加または削除後にもページの並びを維持

バグの修正

  • 修正：.scpagesにて除外されたページのメニュー処理
  • 修正：.scpages掲載ファイルのキャッシュの非一貫性
  • 修正：エディットモードでユーザーはページリンクを介してウェブサイトを移動することができなかった
  • 修正： .scpages掲載のファイルにおけるメニューへのSitecakeの非対応
  • 修正：URLをクエリストリングで変換
  • 修正：h4とh6ツールバーボタンの入れ替え
  • 修正：ページの存在しないメニューの保存不可

2.4.4リリースノート

2017年4月27日

バグの修正

  • キャッシュの非一貫性によりSitecakeが .scpagesファイルを介して除外されたページを削除

2.4.3リリースノート

2017年4月25日

改善

  • UTF-8エンコードのメニューアイテム対応
  • .scpagesにて特定したファイルとフォルダをアスタリスクで宣言

バグの修正

  • 修正：Sitecakeがユーザースタイルの定義されていないコンテンツのクラスを除去
  • 修正：ページ詳細モーダルでの不正確なページURL表示

2.4.2リリースノート

2017年4月21日

改善

  • ページマネージャー追加。既存ページの複製、削除、並び替え。ドラッグ＆ドロップでメニューアイテムを管理
  • 基本的なサイト上SEO（ページタイトル、説明、URLを設定し、Googleスニペットでの表示を確認）
  • PHP7サポートの更新
  • メニューアイテムのデフォルトテンプレートをエレメントとしてレンダリングするよう変更
  • エラーの対処とログの改善
  • 'pages.use_document_relative_paths'のデフォルト値をfalseに変更
  • ページマネージャーキーボードショートカット（D,H,S,ESC, DEL）追加

バグの修正

  • 段落、見出し、画像エレメントの属性がSitecakeにより除去されないように改善
  • Sitecakeユーザースタイルとして定義されていないクラスがSitecakeにより除去されないように改善
  • バックアップハンドリングの修正
  • 5.5からPHPバージョンチェック5.6へ
  • SEOページ情報（ページマネージャー内）を正常に保存
  • ネイティブHP DOMを使ったHTML読み込みの修正（警告の抑制）

2.3.6リリースノート

2016年10月15日

改善

  • PHP 7への対応開始。Sitecake実行の最新ミニマルバージョン（PHP 5.6）
  • リンクを新規ウィンドウで開く機能搭載
  • ページ上でSitecakeを開くリクエストパラメータ名を"scpage"に変更
    (sitecake.php?scpage=contact.html)

バグの修正

  • 修正：BACKSPACE/DELによる箇条書きのコンテンツ削除
  • 修正：CloudFlareキャッシュの問題
  • 修正：サブディレクトリからのログアウト後のBad (404) リダイレクト
  • 修正：テキストエレメント編集中の[元に戻す]実行時のクラッシュ
  • 修正：画像プレースホルダーの削除[x]ボタンのクリック不可
  • 修正：Srcset消失
  • 修正：[元に戻す]操作によるbrタグの複製

2.3.1リリースノート

2016年6月30日

最新機能

  • PHPサイト対応 - HTMLファイル/ページを除く（PHPページのSitecakeを利用した編集も可能 *特定の制限はあり）
  • サブディレクトリ対応 - Sitecakeを利用しサブディレクトリ内のページを編集
  • どのページをSitecake下でコントロールするか厳密に設定
  • 手動でのテンプレート変更後に自動で下書きコンテンツを再読み込み - ページファイルの手動の変更に合わせて、下書きファイルも (sitecake-temp内)自動で更新
  • 最新サーバーサイド (PHP) セッションハンドラー対応(memcache,memcached,redis)
  • Logging サーバーサイド (PHP) のエラーをログファイルに記録

改善

  • 設定可能なエントリポイントURL (sitecake.php)
  • コンテンツのコンテナ内のHTMLコードの整理
  • マルチクラスユーザースタイルに対応
  • 名称のつけられたコンテンツコンテナ内ユーザースタイルに対応
  • エディタ設定ファイルのキャッシュを阻止
  • サーバーサイドのログとエラーハンドリングを改善

バグの修正

  • 修正：擬似クラスのCSSがユーザースタイルとして扱われる
  • 修正：インデックスでないページからのログアウト時の問題
  • 修正：htmlコンテントアイテムが最初の編集の後クリック/選択できない
