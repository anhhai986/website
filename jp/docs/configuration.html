設定

Sitecakeの設定ファイルは/sitecakeフォルダに格納されています。

<sitecake>/
├── <version>/
│ ├── client/
│ └── config/
│ ├── ...
│ └── config.php
├── credentials.php.default
├── credentials.php
└── editor.cnf

設定ファイルの種類は3つです：

  • /sitecake/credentials.php - CMS admin パスワード
  • /sitecake/editor.cnf - エディター/ツールバーの設定
  • /sitecake/<version>/config/config.php - CMS一般の設定

Adminパスワード

credentials.phpファイルにはパスワードのみが含まれています。パスワードはSHA1ハッシュとしてサーバーに保持され、このような見た目です:

<?php $credentials = "d033e22ae348aeb5660fc2140aec35850c4da997"; ?>

デフォルトのパスワードはadminです。Sitecakeログインダイアログのパスワード変更オプションで変更できます。

代わりに、credentials.phpファイルを編集することで、新しいパスワードを手動で設定することもできます。 SHA1ハッシュ（二重引用符で囲まれた文字）を新しいSHA1ハッシュ値に置き換えてください。 たとえば、オンラインのSHA1エンコーダツールを使用して新しいハッシュを取得することができます (例：SHA1 Online)。

パスワードを忘れた場合は、credentials.phpを credentials.php.defaultに置き換えてデフォルトにリセットしてください。

エディターの設定

editor.cnfファイルからはSitecakeのエディターとツールバーの設定が行え、このような見た目になっています：

# Toolbar configuration
# Toolbar.components - the list of content type available on the toolbar
# Possible components:
# HEADING1 - text - H1
# HEADING2 - text - H2
# HEADING3 - text - H3
# HEADING4 - text - H4
# HEADING5 - text - H5
# HEADING6 - text - H6
# TEXT - text
# TEXTLIST - text list
# IMAGE - image
# FLASH - any flash/swf content
# VIDEO - youtube/vimeo/etc. flash/html5 video
# MAP - google map widget
# HTML - row HTML block
# FILE - generic file upload
# SEP - toolbar module separator

Toolbar.components = HEADING1,HEADING2,HEADING3,TEXTLIST,TEXT,SEP,IMAGE,VIDEO,SEP,FLASH,MAP,HTML,FILE

# upload size limit in KB
FileUploaderItem.uploadSizeLimit = 8192

# interface language (en, sl, sr, es, de, fr, dk, it, ru, cs, sk, pt, pt-br, auto)
Locale.code = auto

Toolbar.componentsは、編集を許可したいアイテムのリストです。 デフォルトでは、H1-H3見出しのみが許可されています。 リストは並べ替え、空白（SEP）でカスタマイズすることができます。

FileUploaderItem.uploadSizeLimitは、アップロードの最大画像/ファイルサイズを設定します。 大規模な画像では、8192 KBのデフォルト値が問題になることがあります。その際には自由に変更してください。

Locale.codeはUI言語を設定します。 デフォルトは、エディタのIPアドレスに基づく自動検出です。SitecakeのUIは12言語に翻訳されています。多言語化の詳細についてはこちらをご覧ください。

一般設定

config.phpファイルからは以下のようなCMSの一般的な設定が行えます：

debug = false - Sitecakeがデバッグモードになっていることを示します。 これは、エラーの表示とログの記録に影響します。

entry_point_file_name = sitecake.php - エントリポイントファイルの名前を変更してセキュリティを強化することができます。 名前の変更後には、この設定値を新しいファイル名に設定してください。

session.save_handler = files - サーバ上で使用されるセッションハンドラです。'file'、'memcache'、'memcached'、 'redis'に設定することができます。

session.options - セッションストレージに関するオプションです。 可能なオプションは設定ファイルの中でコメントとして記録されています。 ネイティブのセッションハンドラでこれらのオプションを使用できるようにするには、サーバ上でini_set関数を有効にする必要があります。

log.size = 2MB - ファイルがアーカイブされ、新しいログファイルが作成される前の最大ログサイズを規定します。

log.archive_size = 5 - サーバー上に保管されているアーカイブ済みログファイルの数を表しています。

error.level'] = E_ALL & ~E_DEPRECATED & ~E_STRICT; - PHPエラーレベルを示すものです。

site.default_pages = 'index.html','index.htm','index.php','index.php5' - デフォルトのインデックスページの配列です。 Sitecakeは編集可能なウェブサイトファイルを検索する際にここにある拡張機能を使用します。

pages.prioritize_manual_changes = true - Sitecakeが編集するページを開くたびに、最後のセッションで記録された下書きの編集または、サーバーにある元のHTMLページからの新規作成という2つの異なるソースから開始することができます。 デフォルトでは、このオプションはtrueに設定されています。つまり、HTMLを手動で変更すると、下書きが破棄され、更新されたHTMLから新たに開始されます。

pages.use_document_relative_paths = true - 生成されたページを“ドキュメントルート”または“サイトルート”と相対的にリンクする必要があるかどうかを示しています。このオプションをtrueに設定すると、ナビゲーションリンクのhref値にドキュメント相対パスが使用されます。 それ以外の場合は、サイトの相対パスが使用されます。

filesystem.adapter = local - サーバー上のPHPプロセスにWebサイトルートファイルへの書き込み権限がない場合は、「ftp」アダプタを使用して、必要なFTPアクセスプロパティを提供します。FTPプロトコルは、Webサイトのルートファイルを管理するために使用されます。

その他の選択肢を見るにはconfig.phpを開いてください。
